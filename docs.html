Problem & Solution

YC PITCH
❌ The Problem

Over 3 million students sit SSC/HSC every year with no personalised AI support

Quality coaching costs 3,000–10,000 BDT/month — unaffordable for most families

Global EdTech platforms are English-only — 70% of Bangladeshi students are Bangla-medium

Generic study materials don't adapt to stream (Science/Arts/Commerce) or curriculum level

✅ The Solution

AI tutor powered by Llama 3.3-70B via Groq — responds in Bangla or English

Full personalisation: level (SSC/HSC) + stream (Science/Arts/Commerce) + language injected into every AI prompt

Adaptive MCQ quizzes with instant scoring and automatic weak-area detection

100% free — hosted on Cloudflare Pages, AI on Groq free tier, DB on Supabase free tier

02

Product Overview

🤖

AI Tutor

Groq Llama 3.3-70B answers curriculum questions in a structured 4-step format: Explanation → Formula → Example → Exam Tip. Bilingual support.

📝

Adaptive Quizzes

Subject-wise MCQ with strict no-peek rule. Answers revealed only after submission. Automatic weak-area detection by subject score comparison.

📊

Smart Dashboard

Study tracker, XP system, streak counter, leaderboard and achievement badges. Performance analytics updated in real time.

📚

Resource Library

Formula sheets, notes, and past papers organised by subject, stream and level. Content stored in Supabase for fast retrieval.

🎯

4-Stage Onboarding

Phone + Email → Language → Level → Stream. Every part of the platform adapts automatically to the student's selections.

🇧🇩

Bangladesh-First

Built for Bangladeshi ISP constraints — no Vercel (blocked), Cloudflare Pages works everywhere. Full Bangla support throughout.

API Documentation

Kronexas exposes one internal API endpoint and consumes two external APIs.
Internal Endpoint
POST /chat AI Tutor inference proxy
Accepts student message + context, proxies to Groq API, returns structured response. API key never exposed to browser. 
// Request body
{
  "messages": [...],        // last 8 messages
  "system": "You are Kronexas, an AI tutor for SSC Science...",
  "model": "llama-3.3-70b-versatile"
}

// Response
{
  "content": [{ "type": "text", "text": "..." }]
}
External APIs Consumed
POST api.groq.com/openai/v1/chat/completions Groq LLM
Auth: Bearer token (GROQ_API_KEY env var). Model: llama-3.3-70b-versatile. Max tokens: 1000. Temperature: 0.7.
POST lvfghdezyopklgossuam.supabase.co/rest/v1/users Supabase DB
Auth: Supabase anon key (public). Inserts phone + email on user login. RLS disabled for public insert.
08

AI Layer

CORE

Model

Llama 3.3-70B Versatile — open-weight Meta model via Groq's LPU inference. Selected for speed, cost (free), and instruction-following quality.

Personalisation

Every request injects level + stream + language into the system prompt. The model adapts vocabulary, curriculum scope, and response language automatically.

Response Format

Enforced 4-step structure: Explanation → Formula → Example → Exam Tip. Custom fmtAI() function parses markdown to HTML before display.

Token Optimisation

3 controls: max_tokens: 1000 hard cap, slice(-8) context window, dense system prompt. Keeps usage within Groq free tier.

RAG (Planned)

Supabase contents table stores 18+ SSC/HSC curriculum entries. Next phase: pgvector embeddings for semantic retrieval injected into prompts.

Guardrails

System prompt restricts AI to educational topics only. No PII sent to Groq. API key server-side only. Off-topic queries politely redirected.

09

Security

API Key Protection

GROQ_API_KEY stored in Cloudflare environment variables. Never in client code, never in GitHub repo.

No Password Storage

Zero passwords stored anywhere. Phone + email only, held in JS session memory — reset on page refresh.

No PII to AI

Only question text and academic context forwarded to Groq. Names, phone, email never sent to any AI model.

Input Sanitisation

.trim() on all inputs, @ validation on email, format check on phone before any processing or DB write.
